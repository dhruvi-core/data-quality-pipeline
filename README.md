# Trading Firm Data Quality Pipeline

## Overview

This project builds a Data Quality (DQ) Pipeline for a Trading Firm.

The pipeline:
- Reads trading data from CSV files
- Standardizes the data using Apache Spark
- Performs Data Quality (DQ) validations
- Stores valid data and invalid data separately in MongoDB
- Uses Apache Airflow to automate the workflow

## Technologies

- Python
- Apache Spark (PySpark)
- Apache Airflow
- MongoDB
- Docker
- Git
